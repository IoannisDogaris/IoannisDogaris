# Retail Analytics SQL + R (PostgreSQL)

This project is a complete, portfolio‑ready retail analytics pipeline built using PostgreSQL for data modeling and SQL analysis, and R for exploratory analysis, visualization, and reporting. It simulates a small retailer selling coffee, tea, snacks, and breakfast items across multiple regions and sales channels.

The goal is to demonstrate practical, real‑world data analytics skills: database design, SQL transformations, business‑oriented queries, and integrated R workflows.

## ▶️ How to Run the Project

### 1. Create the PostgreSQL Database

CREATE DATABASE retail_analytics;

### 2. Clone the Repository

git clone https://github.com/IoannisDogaris/retail-analytics-sql.git  
cd retail-analytics-sql

### 3. Configure PostgreSQL Access in R

Edit the connection block in `retail_analysis.Rmd`:
```
con <- dbConnect(  
  RPostgres::Postgres(),  
  dbname = "retail_analytics",  
  host   = "localhost",  
  port   = 5432,  
  user   = "postgres",  
  password = "postgres"  
)
```
### 4. Install Required R Packages
```
install.packages(c(  
  "tidyverse",  
  "DBI",  
  "RPostgres",  
  "dbplyr",  
  "here"  
))
```
### 5. Run the Notebook

- Open `r-notebook/retail_analysis.Rmd`  
- Knit to HTML or run interactively  

The notebook will:  
- Reset the schema  
- Load the CSV data  
- Create all views  
- Run EDA + visualizations  
- Output insights
